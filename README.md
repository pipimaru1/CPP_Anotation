## 1. 概要

`CPP_Anotation6` は、YOLOシリーズの学習データを作成するためのアノテーション支援アプリケーションです。C++ / Win32 API / GDI+ で実装されており、画像フォルダ内の画像を高速に読み込み、YOLO形式のアノテーションファイルを作成・編集・保存できます。画像は全てメモリに格納されるので、超高速に動作します。

学習済みONNXモデルを使った推論にも対応しており、学習済みモデルの簡易検証、自動アノテーション、推論結果の手動確定が可能です。矩形の追加・編集・削除、未アノテーション画像へのジャンプ、小さすぎるラベルの検出・補正、余裕幅を付けたラベル保存など、YOLO学習データ作成に必要な実務向け機能を備えています。

本書では、最終バージョンである `CPP_Anotation6` プロジェクトのみを説明します。

## 2. 主な機能

- 画像フォルダ内の画像を一覧化して順番に表示
- YOLO形式のラベルファイルを読み込み・保存
- マウス操作によるアノテーションボックス作成
- 既存ボックスのクラス変更、サイズ変更、削除
- クラス定義ファイルによるラベル名・色・線種・線幅の変更
- 学習済みONNXモデルによる自動アノテーション
- YOLOv5 / YOLOv7 系、YOLOv8 / YOLOv11 系のモデル切り替え
- ONNXモデル、クラス定義ファイル、解像度を `default.ini` からメニュー化
- 推論結果の右クリック確定、または閾値指定による一括確定
- 未アノテーション画像へのジャンプ
- 小さすぎる、または不適切なアノテーションボックスへのジャンプ
- 小さすぎるボックスの補正
- ラベル保存時の余裕幅付与
- 小さすぎるラベルを除外して保存
- 画像とラベルの削除、実体は `deleted` フォルダへの移動
- OpenMPを使った画像・ラベルの並列読み込み

## 4. 起動方法

### 4.1 通常起動

ビルド後に生成された `CPP_Anotation6.exe` を実行します。

```bat
CPP_Anotation6.exe
```

引数なしで起動した場合、ソース上では初期画像フォルダとして次の相対パスを参照します。

```text
../images/
```

また、起動時にカレントディレクトリの `default.ini` を探し、存在する場合はONNXモデル選択メニューを作成します。

そのため、配布または単体実行する場合は、次のいずれかの方法を推奨します。

1. `CPP_Anotation6.exe` と同じフォルダに `default.ini` を置く
2. Visual Studioから実行する場合は、作業ディレクトリに `default.ini` が存在するようにする
3. `default.ini` 内で指定しているONNXファイルとクラス定義ファイルのパスを、実際の環境に合わせる

### 4.2 コマンドライン引数付き起動

画像フォルダ、ラベルフォルダ、クラス定義ファイルを指定して起動できます。

```bat
CPP_Anotation6.exe <分類ファイル> <画像フォルダ> <ラベルフォルダ> [ONNXファイル] [ONNX入力解像度]
```

引数の意味は次のとおりです。

| 引数 | 必須 | 説明 |
|---|---:|---|
| `<分類ファイル>` | 必須 | クラス定義ファイル。例: `TimeStudyA.txt`, `awz.txt`, `coco80.txt` |
| `<画像フォルダ>` | 必須 | アノテーション対象画像のフォルダ |
| `<ラベルフォルダ>` | 必須 | YOLO形式ラベルファイルの読み込み・保存先フォルダ |
| `[ONNXファイル]` | 任意 | 自動アノテーションに使用するONNXモデル |
| `[ONNX入力解像度]` | 任意 | `640`, `1280`, `1920` のいずれか |

例:

```bat
CPP_Anotation6.exe C:\Programming\CPP_Anotation\TimeStudyA.txt C:\dataset\images C:\dataset\labels C:\models\yolov8x.onnx 1280
```

この形式で起動した場合、起動時に次の処理が行われます。

- クラス定義ファイルを読み込む
- 画像フォルダを読み込む
- ラベルフォルダ内の `.txt` ラベルを読み込む
- ONNXファイルが指定されていれば自動アノテーション用モデルとして設定する
- ONNX入力解像度が指定されていれば推論解像度として設定する

指定したファイルまたはフォルダが存在しない場合は、起動後にエラーメッセージが表示されます。

## 5. 設定ファイル `default.ini`

### 5.1 役割

`default.ini` は、自動アノテーションで使用するONNXモデルのプリセット一覧を定義する設定ファイルです。起動時に読み込まれ、メニューの `自動アノテーション` → `ONNXファイル選択` に反映されます。

ソースでは、起動時に次のファイル名をカレントディレクトリから読み込む実装です。

```cpp
std::wstring _inifile = L"default.ini";
```

Visual Studioから実行する場合と、ビルド後のEXEを直接実行する場合でカレントディレクトリが異なることがあります。メニューにONNXプリセットが出ない場合は、実行時のカレントディレクトリに `default.ini` があるか確認してください。

### 5.2 文字コード

`CPP_Menu.cpp` では `default.ini` をCP932、いわゆるShift-JISとして読み込む処理が使われています。

日本語のモデル名を使う場合は、`default.ini` をShift-JISで保存するのが安全です。UTF-8を使う場合は、環境によって文字化けしないか確認してください。

### 5.3 書式

1行に1つのONNXモデルプリセットを記述します。

```text
ONNX[番号], メニュー表示名, ONNXファイルパス, クラス定義ファイルパス, YOLO種別, 入力解像度, クラス数
```

例:

```text
ONNX[0], COCO_V5Sモデル, C:/Programming/CPP_Anotation/yolov5s.onnx, C:\Programming\CPP_Anotation\coco80.txt, v5, 640, 80
ONNX[1], COCO_V8Xモデル, C:/Programming/CPP_Anotation/yolov8x.onnx, C:\Programming\CPP_Anotation\coco80.txt, v8, 640, 80
ONNX[2], 工場物体認識_YOLOV5X6, C:\Programming\CPP_Anotation\AWZ.onnx, C:\Programming\CPP_Anotation\awz.txt, v5, 1280, 11
```

各項目の意味は次のとおりです。

| 項目 | 例 | 説明 |
|---|---|---|
| `ONNX[番号]` | `ONNX[0]` | 人が見やすくするためのID。実処理では主に行順が使われます |
| メニュー表示名 | `COCO_V8Xモデル` | アプリのONNX選択メニューに表示される名前 |
| ONNXファイルパス | `C:\models\yolov8x.onnx` | 推論に使うONNXファイル |
| クラス定義ファイルパス | `C:\models\coco80.txt` | クラス名、色、線種、線幅を定義するファイル |
| YOLO種別 | `v5`, `v8`, `v11` | YOLO出力形式の切り替えに使用 |
| 入力解像度 | `640`, `1280`, `1920` | ONNXモデルの入力サイズ |
| クラス数 | `80`, `11` | モデルのクラス数。設定情報として保持されます |

`#` で始まる行と空行は無視されます。カンマ区切りでパースされるため、パスや表示名にカンマを含める場合は避けてください。

### 5.4 プリセット選択時の動作

`default.ini` に定義したプリセットをメニューから選択すると、次の値が一括で設定されます。

- ONNXファイルパス
- YOLOバージョン
- 入力解像度
- クラス定義ファイル

クラス定義ファイルが存在する場合は、その場でクラス一覧、色、線種、線幅が更新されます。ファイルが存在しない場合は警告が表示されます。

## 6. クラス定義ファイル

### 6.1 役割

クラス定義ファイルは、アノテーション対象のクラス名と描画スタイルを定義します。

例:

```text
0, Work, 255, 255, 0, DashStyleDash, 2
1, SubWork, 255, 0, 0, DashStyleSolid, 2
2, Bring_Handling_Parts, 0, 255, 0, DashStyleSolid, 3
3, Move, 0, 0, 255, DashStyleDash, 2
```

### 6.2 書式

```text
クラスID, クラス名, R, G, B, 線種, 線幅
```

| 項目 | 説明 |
|---|---|
| クラスID | YOLOラベルに保存されるクラス番号 |
| クラス名 | 画面やポップアップメニューに表示する名前 |
| R/G/B | ボックス描画色。0〜255 |
| 線種 | `DashStyleSolid`, `DashStyleDash`, `DashStyleDot`, `DashStyleDashDot`, `DashStyleDashDotDot` など |
| 線幅 | ボックス描画時のペン幅 |

このファイルを切り替えることで、作業認識、工場物体認識、COCOモデルなど、用途別のラベルセットを使い分けできます。

## 7. ラベルファイル形式

保存されるラベルファイルはYOLO形式です。画像ファイルと同じベース名の `.txt` ファイルとして保存されます。

例:

```text
0 0.512345 0.423456 0.120000 0.180000
3 0.300000 0.250000 0.080000 0.140000
```

各行の意味は次のとおりです。

```text
class_id x_center y_center width height
```

座標はすべて画像サイズに対する正規化値です。

| 項目 | 説明 |
|---|---|
| `class_id` | クラス番号 |
| `x_center` | ボックス中心X座標、0.0〜1.0 |
| `y_center` | ボックス中心Y座標、0.0〜1.0 |
| `width` | ボックス幅、0.0〜1.0 |
| `height` | ボックス高さ、0.0〜1.0 |

アプリ内部では左上座標と幅・高さで矩形を扱いますが、読み書き時にはYOLO形式へ変換されます。

## 8. 基本操作

### 8.1 作業開始手順

1. アプリを起動する
2. `ファイル` → `クラス分類リスト読込` でクラス定義ファイルを読み込む
3. `ファイル` → `画像フォルダを選択` で画像フォルダを選択する
4. 必要に応じて `ファイル` → `アノテーションデータを読込` で既存ラベルフォルダを選択する
5. 画像を確認しながらアノテーションを追加・修正する
6. `ファイル` → `アノテーションデータを保存` で保存する

コマンドライン引数で分類ファイル、画像フォルダ、ラベルフォルダを指定して起動した場合は、手順2〜4を省略できます。

### 8.2 アノテーションボックスの作成

1. 画像上でマウス左ボタンを押す
2. ドラッグして矩形を作成する
3. マウス左ボタンを離す
4. クラス選択ポップアップが表示される
5. クラスを選択すると、ボックスが登録される

クラスを選択せずキャンセルした場合は、ボックスは登録されません。

### 8.3 ボックスの編集

既存ボックスの辺または角にマウスを合わせると、そのボックスが編集対象になります。

- 左ドラッグ: ボックスの辺または角を移動してサイズを変更
- 右クリック: クラス変更、補正、削除などの編集メニューを表示
- `DELETE` メニュー: 選択中のボックスを削除
- `FIX` メニュー: 小さすぎるボックスを最小サイズ以上に補正

編集された画像は、画像移動時や保存時にラベルファイルへ反映されます。

### 8.4 画像の移動

| 操作 | 内容 |
|---|---|
| `A` または `←` | 前の画像へ移動 |
| `D` または `→` | 次の画像へ移動 |
| `Shift + ←/→` | 10枚単位で移動 |
| `Ctrl + ←/→` | 100枚単位で移動 |
| `Ctrl + Alt + D` または `Ctrl + Alt + →` | 次の未アノテーション画像へ移動 |

画像を移動する前に、現在画像が編集済みであれば単体ラベル保存が実行されます。

### 8.5 表示操作

| 操作 | 内容 |
|---|---|
| マウスホイール | ズームイン / ズームアウト |
| マウス中ボタンドラッグ | 画像のパン移動 |
| `Q` | バウンディングボックス表示のON/OFF |
| `S` | 現在画像と次画像の比較表示を切り替え |

比較表示はタイマーで現在画像と次画像を切り替えて表示します。連続画像の差分確認に利用できます。

### 8.6 コピー・クリア・Undo

| 操作 | 内容 |
|---|---|
| `C` または `V` | 前の画像のラベルを現在画像へコピー |
| `Ctrl + 0` | 現在画像のラベルをすべてクリア |
| `Ctrl + Z` | 直前のクリアまたは削除をUndo |

前画像コピーでは確認ダイアログが表示されます。

## 9. メニュー操作

### 9.1 ファイルメニュー

| メニュー | 内容 |
|---|---|
| `クラス分類リスト読込` | クラス定義ファイルを読み込む |
| `画像フォルダを選択` | アノテーション対象画像フォルダを読み込む |
| `アノテーションデータを読込` | YOLO形式ラベルファイルのフォルダを読み込む |
| `アノテーションデータを保存` | ラベルファイルを保存する |
| `余裕幅を付けて保存する` | 13%, 25%, 50%, 75%, 100%の余裕幅を付けて保存する |
| `小さいラベルを保存しない` | 指定サイズ以下のラベルを保存対象から除外する |
| `小さいオブジェクトを補正して保存する` | 小さいラベルを除外せず、最小サイズへ補正して保存する |
| `終了する` | アプリを終了する |

### 9.2 余裕幅を付けて保存

アノテーションボックスを拡大して保存できます。

| メニュー | 保存倍率 |
|---|---:|
| `13%余裕幅で保存` | 1.13倍 |
| `25%余裕幅で保存` | 1.25倍 |
| `50%余裕幅で保存` | 1.50倍 |
| `75%余裕幅で保存` | 1.75倍 |
| `100%余裕幅で保存` | 2.00倍 |

余裕幅を付けた保存は、元のラベルを上書きしないよう、通常とは別の保存先フォルダを選択することを推奨します。

### 9.3 小さいラベルを保存しない

ラベル保存時に、画面比で小さすぎるラベルを除外できます。

| メニュー | 動作 |
|---|---|
| `全て保存` | すべてのラベルを保存 |
| `画面比1%以下を保存しない` | 幅または高さが1%未満のラベルを除外 |
| `画面比1.5%以下を保存しない` | 幅または高さが1.5%未満のラベルを除外 |
| `画面比2%以下を保存しない` | 幅または高さが2%未満のラベルを除外 |
| `画面比3%以下を保存しない` | 幅または高さが3%未満のラベルを除外 |
| `画面比5%以下を保存しない` | 幅または高さが5%未満のラベルを除外 |
| `画面比10%以下を保存しない` | 幅または高さが10%未満のラベルを除外 |

この設定は保存後にリセットされます。

### 9.4 編集メニュー

| メニュー | 内容 |
|---|---|
| `不適切ボックスへジャンプ` | 小さすぎるボックスを含む画像へジャンプ |
| `未アノテーション画像へジャンプ` | ラベルがない画像へジャンプ |
| `現在の画像の異常ラベルを修正` | 現在画像の小さすぎるラベルを補正 |
| `全ての画像の異常ラベルを修正` | 全画像の小さすぎるラベルを補正し、保存処理へ進む |
| `現在の画像のラベルをクリア` | 現在画像の全ラベルを削除 |
| `削除を取り消し Undo` | 直前の削除・クリアを復元 |
| `画像の削除` | 現在の画像とラベルを `deleted` フォルダへ移動 |

不適切ボックスの判定には、ONNX入力解像度と最小サイズ設定が使われます。初期値では最小幅・高さが8ピクセル相当として判定されます。

### 9.5 ラベル目次

`ラベル目次` メニューでは、現在画像に含まれるラベル一覧をポップアップ表示できます。ラベルを選択すると、対象ラベルのクラス変更や削除などの編集メニューを表示できます。

## 10. 自動アノテーション

### 10.1 基本手順

1. `自動アノテーション` → `ONNXファイル選択` から使用するONNXプリセットを選択する
2. 必要に応じて `YOLOバージョン`、`解像度`、`ONNXファイル選択` を調整する
3. `AI実行` を選択する
4. 現在画像に対して推論が実行され、候補ボックスが表示される
5. 候補ボックスを確認し、右クリックで確定またはクラス変更する

`default.ini` にプリセットを登録しておくと、ONNXファイル、クラス定義ファイル、YOLO種別、解像度をまとめて切り替えられます。

### 10.2 推論候補の確定

自動アノテーションで表示された候補ボックスは、通常のラベルとは別の一時候補として表示されます。

候補ボックス上で右クリックすると、次の操作ができます。

- `確定`: 推論結果のクラスのまま本ラベルとして登録
- クラス名選択: クラスを変更して本ラベルとして登録
- `DELETE`: 候補またはラベルを削除
- `CANCEL`: 操作をキャンセル

確定された候補は、通常のアノテーションボックスとして扱われます。

### 10.3 感度設定

メニューの `高感度設定` では、推論の信頼度閾値を変更できます。

| メニュー | confThreshold |
|---|---:|
| `高感度` | 0.01 |
| `超高感度` | 0.005 |
| `超々高感度` | 0.001 |
| `超々々高感度 for V8,11` | 0.0 |

閾値を下げるほど候補ボックスは増えますが、誤検出も増えます。自動アノテーションでは、最初は高めの閾値で確認し、検出漏れが多い場合に下げる運用を推奨します。

### 10.4 キーボードによる自動アノテーション

数字キー `1`〜`7` は、自動アノテーション実行のショートカットとして使われます。各キーに対応する閾値は次のとおりです。
数字キーは認識するだけです。`Ctrl + 1`〜`Ctrl + 7`キーで確定します。

| キー | confThreshold |
|---|---:|
| `1` | 0.25 |
| `2` | 0.10 |
| `3` | 0.01 |
| `4` | 0.001 |
| `5` | 0.0001 |
| `6` | 0.000005 |
| `7` | 0.0000005 |

`Ctrl + 1`〜`Ctrl + 7` は、表示中の推論候補のうち、対応する信頼度閾値以上のものを一括でします。

### 10.5 プリアノテーション感度調整

`自動アノテーション` → `プリアノテーション感度調整` を選択すると、`NMS` と `Conf` のスライダーが表示されます。

| 項目 | 説明 |
|---|---|
| `Conf` | 推論候補として採用する最低信頼度 |
| `NMS` | 重複するボックスを統合・除外するための閾値 |

`Conf` を下げると検出候補が増え、`NMS` を調整すると重複ボックスの残り方が変わります。

## 11. 削除機能

`編集` → `画像の削除` を実行すると、現在の画像と対応するラベルファイルを削除扱いにできます。

実装上は完全削除ではなく、元フォルダの親フォルダ配下に `deleted` フォルダを挿入した場所へ移動します。

例:

```text
C:\dataset\images\FILE0001.jpg
C:\dataset\labels\FILE0001.txt
```

移動後:

```text
C:\dataset\deleted\images\FILE0001.jpg
C:\dataset\deleted\labels\FILE0001.txt
```

注意点として、ソース上では移動先の画像拡張子が `.jpg` 固定で生成されています。`.png` や `.bmp` などを扱う場合は、移動時の拡張子処理を必要に応じて修正してください。

## 12. 作業時の注意点

### 12.1 画像名とラベル名

画像とラベルは同じベース名で対応します。

```text
images/FILE0001.jpg
labels/FILE0001.txt
```

ラベル読み込み時は、画像ファイル名に対応する `.txt` ファイルがラベルフォルダに存在するか確認されます。

### 12.2 保存先の上書き

通常保存、余裕幅付き保存、小さいラベル除外保存は、いずれも選択した保存先フォルダに `.txt` ファイルを書き込みます。

余裕幅付き保存や小さいラベル除外保存は元データを破壊しやすいため、通常のラベルフォルダとは別フォルダへ保存することを推奨します。

### 12.3 `default.ini` の配置

`default.ini` は起動時のカレントディレクトリから読み込まれます。

メニューにONNXプリセットが出ない場合は、次を確認してください。

- 実行中のカレントディレクトリに `default.ini` があるか
- `default.ini` の文字コードがShift-JISになっているか
- ONNXファイルパスが存在するか
- クラス定義ファイルパスが存在するか

### 12.4 ONNXモデルと解像度

ONNXモデルの入力解像度と、アプリ側の設定解像度が一致していないと、推論結果や小さいラベル補正の判定が期待通りにならない場合があります。

`default.ini` でモデルごとに `640`, `1280`, `1920` などの解像度を正しく指定してください。

## 13. ビルド方法

### 13.1 開発環境

`CPP_Anotation6` はVisual Studio用のC++プロジェクトです。

確認できる主なビルド条件は次のとおりです。

| 項目 | 内容 |
|---|---|
| IDE | Visual Studio 2022想定 |
| Platform Toolset | `v143` |
| Windows SDK | `10.0` |
| 文字セット | Unicode |
| C++標準 | C++17 |
| OpenMP | 有効 |
| 主要ライブラリ | Win32 API, GDI+, OpenCV DNN |

### 13.2 必要な外部ライブラリ

OpenCVが必要です。プロジェクトファイルでは、OpenCV 4.12.0系の `opencv_world4120.lib` / `opencv_world4120d.lib` を参照しています。

使用されている環境変数は次のとおりです。

| 環境変数 | 用途 |
|---|---|
| `OPENCV_CPU_INC` | OpenCVのincludeディレクトリ |
| `OPENCV_CPU_BIN` | CPU版OpenCVのビルド/インストールディレクトリ |
| `OPENCV_GPU_BIN` | CUDA対応OpenCVのビルド/インストールディレクトリ |

代表例:

```bat
set OPENCV_CPU_INC=C:\opencv\build\install\include
set OPENCV_CPU_BIN=C:\opencv\build\install\x64\vc17
set OPENCV_GPU_BIN=C:\opencv_gpu\build\install\x64\vc17
```

実際のパスは、自分のOpenCVビルド環境に合わせて変更してください。

### 13.3 Visual Studioでのビルド

1. `CPP_Anotation.sln` をVisual Studioで開く
2. ソリューションエクスプローラーで `CPP_Anotation6` を選択する
3. 必要に応じて `スタートアッププロジェクトに設定` を行う
4. 構成を選択する
   - `Release|x64`: CPU版OpenCVでのリリースビルド
   - `Debug|x64`: CPU版OpenCVでのデバッグビルド
   - `Release_GPU|x64`: GPU版を想定したリリースビルド
   - `Debug_GPU|x64`: GPU版を想定したデバッグビルド
5. ビルドを実行する

通常利用では `Release|x64` を推奨します。GPUでOpenCV DNNを使用したい場合は、CUDA対応でビルドされたOpenCVと、`Release_GPU|x64` または `Debug_GPU|x64` を使用します。

### 13.4 コマンドラインでのビルド例

Visual Studioの開発者コマンドプロンプトから、次のようにビルドできます。

```bat
msbuild CPP_Anotation.sln /t:CPP_Anotation6 /p:Configuration=Release /p:Platform=x64
```

GPU構成の場合:

```bat
msbuild CPP_Anotation.sln /t:CPP_Anotation6 /p:Configuration=Release_GPU /p:Platform=x64
```

### 13.5 ビルド後イベント

x64構成では、ビルド後イベントでOpenCVのDLLを出力先にコピーします。

例:

```bat
XCOPY /D /Y $(OPENCV_CPU_BIN)\bin\opencv_world4120.dll $(OutDir)
```

デバッグ構成では `opencv_world4120d.dll`、リリース構成では `opencv_world4120.dll` がコピーされます。

### 13.6 GPU構成の注意点

GPU構成では `_USEGPU` が定義され、推論時にOpenCV DNNのCUDAバックエンドを使用する設定になります。

```cpp
GDNNP.opt.backend = cv::dnn::DNN_BACKEND_CUDA;
GDNNP.opt.target  = cv::dnn::DNN_TARGET_CUDA;
```

GPU構成を使うには、CUDA対応でビルドされたOpenCVが必要です。

なお、`CPP_Anotation6.vcxproj` を確認すると、`Release_GPU|x64` のリンク設定では `$(OPENCV_CPU_BIN)\lib\opencv_world4120.lib` を参照し、ビルド後イベントでは `$(OPENCV_GPU_BIN)\bin\opencv_world4120.dll` をコピーする設定になっています。GPU版OpenCVで統一したい場合は、`Release_GPU|x64` の追加依存ファイルも `$(OPENCV_GPU_BIN)` 側に合わせることを推奨します。

修正例:

```xml
<AdditionalDependencies>$(OPENCV_GPU_BIN)\lib\opencv_world4120.lib;%(AdditionalDependencies)</AdditionalDependencies>
```

### 13.7 実行ファイル配布時の同梱物

ビルドしたEXEを単体配布する場合は、少なくとも次を同じフォルダまたは参照可能な場所に配置してください。

- `CPP_Anotation6.exe`
- `opencv_world4120.dll` または `opencv_world4120d.dll`
- `default.ini`
- 使用するONNXモデルファイル
- 使用するクラス定義ファイル
- 画像フォルダ
- ラベルフォルダ

`default.ini` 内のパスが絶対パスの場合、配布先PCでも同じパスにファイルを配置するか、配布先に合わせて `default.ini` を編集してください。

## 14. トラブルシュート

### ONNXメニューが表示されない

- `default.ini` がカレントディレクトリに存在するか確認する
- `default.ini` の文字コードをShift-JISにする
- 行の項目数が不足していないか確認する
- 入力解像度が数値になっているか確認する

### 自動アノテーションを実行しても結果が出ない

- ONNXファイルが存在するか確認する
- `default.ini` のYOLO種別がモデルに合っているか確認する
- 入力解像度がモデルに合っているか確認する
- `Conf` が高すぎないか確認する
- クラス定義ファイルとモデルのクラス数・クラス順が一致しているか確認する

### ラベルが読み込まれない

- 画像ファイル名とラベルファイル名のベース名が一致しているか確認する
- ラベルファイルがYOLO形式か確認する
- ラベルフォルダを正しく選択しているか確認する

### ビルド時にOpenCVが見つからない

- `OPENCV_CPU_INC` が正しいincludeディレクトリを指しているか確認する
- `OPENCV_CPU_BIN` または `OPENCV_GPU_BIN` が正しいOpenCVディレクトリを指しているか確認する
- `opencv_world4120.lib` または `opencv_world4120d.lib` が存在するか確認する
- DLLが実行ファイルの出力先にコピーされているか確認する

### GPU構成で推論できない

- CUDA対応でOpenCVをビルドしているか確認する
- `OPENCV_GPU_BIN` がCUDA対応OpenCVを指しているか確認する
- `Release_GPU|x64` のリンク先とDLLコピー先が一致しているか確認する
- CUDAランタイムDLLがPATH上にあるか確認する
